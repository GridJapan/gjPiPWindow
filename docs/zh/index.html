什么是 PiP？

PiP（Picture-in-Picture）是一种把画面放进小型悬浮窗口并置顶显示的方式。它和 YouTube 的小窗播放是同一类机制，只是 gjPiP 悬浮显示的不是视频，而是显示器本身。

01

把主显示器放进 PiP，会出现无限镜像

Live capture, not a recording.

PiP（Picture-in-Picture）里的内容是实时捕获。被捕获的屏幕里也包含它自己，所以拖动窗口时，整条镜像隧道都会一起移动。

原速
02

在 PiP 内点击两次 → 光标切过去

Interaction mode.

第一次点击用于聚焦，第二次点击进入操作模式。光标会移动到另一块显示器上，物理鼠标也会直接控制那边。把指针移到窗口边缘，就能回到真实屏幕。万一需要脱身，连续按 5 次 Esc 可强制退出。

1.3倍速 ／ 画面中是没有物理显示器的虚拟显示器（1280×720）
03

一键把所有 PiP 收回主显示器

Gather.

即使 PiP 跑到屏幕外或虚拟显示器上找不到，也可以通过菜单中的“将所有 PiP 收集到主显示器”把它们叫回来。

1.4倍速
04

为每个 PiP 单独叠加 17 种视频滤镜

Core Image, chained.

基础调整（亮度、对比度、饱和度、色相、色温、伽马）、锐化与模糊，以及特殊效果（反色、棕褐色、黑白、暗角、边缘检测、结晶、六边形像素化）。支持多种同时应用，关闭后即可恢复原样。滤镜通过 GPU 应用，捕获链路仍保持零拷贝。

05

功能

同时打开多个 PiP

每块显示器都可以打开一个 PiP。每个 PiP 都是独立捕获。

每个 PiP 独立设置

置顶、帧率、退出边缘和滤镜，都可以按窗口分别设置。

始终置顶

默认OFF。开启后不会被其他窗口遮挡，但也不会出现在 Mission Control 中。

只要移动到屏幕边缘，就能回到真实屏幕

退出操作模式的动作，就像把鼠标移出一块显示器一样。上下左右四条边都可单独ON/OFF，默认只有下边OFF（用于呼出 Dock）。

帧率

每个 PiP 可单独切换 30/60/120 fps。

强制退出

连续按 5 次 Esc，或按 Control + Command + Esc，随时取回鼠标。

07

安装

请从源码构建。TCC（屏幕录制、辅助功能）只会向已签名的 .app 包授予权限，因此请务必使用 build.sh。

# 创建自签名证书（仅首次需要。之后重新构建也不会丢失权限）
./make-signing-cert.sh

# 构建并启动
./build.sh
open build/gjPiP.app
权限用途设置位置
屏幕录制捕获显示器画面隐私与安全性 → 屏幕录制与系统音频录制
辅助功能操作模式（鼠标交接）隐私与安全性 → 辅助功能

两项权限都需要在允许后重启应用。权限弹窗可能会出现在另一块显示器上。

08

与 FreeDisplay 搭配使用

虚拟显示器本身由 GridJapan/FreeDisplay 创建。这是一个 fork，修复了官方 FreeDisplay 现存的 4 个 bug（DDC 崩溃、菜单变空、主显示器设置回滚、无法创建第 3 台显示器），并会记住虚拟显示器的解像度。gjPiP 负责显示并操控那块看不见的屏幕。
